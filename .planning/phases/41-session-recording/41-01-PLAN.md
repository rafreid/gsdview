# Phase 41: Session Recording - Plan

## Goal
User can record, replay, and export GSD sessions for review and sharing.

## Requirements Addressed
- SESS-01: Start/stop session recording via prominent record button
- SESS-02: Capture all file operations with timestamps during recording
- SESS-03: Playback at variable speeds (1x, 2x, 4x, 8x)
- SESS-04: Export session as markdown report
- SESS-05: View list of saved recordings and select one to replay

## Implementation Tasks

### Task 1: Create Session Recorder Module
- Create `src/renderer/session-recorder.js`
- Recording state management (start/stop/toggle)
- Session data structure with operations array
- LocalStorage persistence for recordings

### Task 2: Implement Recording Functionality
- Capture operations with timestamps
- Track relative time from session start
- Store session name, start/end times

### Task 3: Implement Playback
- Load session from recordings list
- Play operations in sequence with timing
- Variable speed support (1x, 2x, 4x, 8x)
- Stop/resume playback

### Task 4: Implement Export
- Generate markdown report from session
- Include operation summary table
- Include files touched table
- Include timeline with timestamps
- Download as .md file

### Task 5: Session Manager Panel
- Modal dialog listing saved recordings
- Play, Export, Delete buttons per recording
- Recording status indicator
- Start/Stop toggle button
- Speed selector

### Task 6: Toolbar Integration
- Add Record button with visual indicator
- Add Sessions button to open manager
- Pulse animation when recording

## Verification
1. Click Record - button changes to "Stop", shows recording state
2. Make file changes - operations captured
3. Click Stop - session saved to list
4. Click Sessions - see recording in list
5. Click Play - operations replay with timing
6. Click Export - markdown file downloads
